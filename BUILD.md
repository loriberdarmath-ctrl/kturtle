# Building KTurtle

The same React + Vite codebase ships as:

- **Web** — single-file HTML on GitHub Pages
- **Desktop** — Windows `.exe` / `.msi`, Linux `.AppImage` / `.deb` via **Tauri**
- **Android** — installable `.apk` / `.aab` via **Capacitor**

All three flavours run **fully offline** after install. No network is ever required, no fonts or icons are fetched at runtime, and the Android build doesn't even request `INTERNET` permission.

---

## Prerequisites

| What you're building | What you need |
|---|---|
| Web only | Node 18+ |
| Desktop (Windows) | Node 18+, [Rust 1.77+](https://www.rust-lang.org/tools/install), [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/) (pre-installed on Win10+ since 2020), Microsoft Visual C++ Build Tools |
| Desktop (Linux) | Node 18+, Rust 1.77+, `libwebkit2gtk-4.1-dev`, `build-essential`, `libssl-dev`, `libayatana-appindicator3-dev`, `librsvg2-dev` |
| Android | Node 18+, JDK 21, [Android Studio](https://developer.android.com/studio) (installs Android SDK + Gradle), `ANDROID_HOME` env var set |

First-time setup (any platform):

```bash
npm install
```

---

## Web build

```bash
npm run build        # writes dist/index.html  (single-file, ~400 KB)
npm run preview      # serves dist/ at http://localhost:4173
```

Upload `dist/` to any static host. GitHub Pages, Netlify, Vercel, an S3 bucket, a flash drive — whatever.

## Desktop (Tauri)

### Development

```bash
npm run tauri:dev
```

This spins up Vite on `:1420`, launches a native window pointed at it, and hot-reloads your React edits live. First launch is slow while Rust compiles — subsequent launches are instant.

### Production installers

```bash
npm run tauri:build
```

Outputs land under `src-tauri/target/release/bundle/`:

- `nsis/KTurtle_0.1.0_x64-setup.exe` — Windows installer (~4 MB)
- `deb/KTurtle_0.1.0_amd64.deb` — Debian/Ubuntu
- `appimage/KTurtle_0.1.0_amd64.AppImage` — portable Linux binary

The Windows installer is **per-user** by default (no admin required). Change `installMode` in `src-tauri/tauri.conf.json` to `perMachine` for system-wide installs.

### Cross-compiling

Tauri only cross-builds between same-arch targets. In practice, build Windows `.exe` from Windows and Linux artefacts from Linux. CI with matrix runners is the easy path; see `.github/workflows/release.yml` if/when added.

## Android (Capacitor)

### One-time setup

If you've never built for Android on this machine, install Android Studio, open it once so it downloads the SDK, then either set `ANDROID_HOME` to the SDK path (typical: `%LOCALAPPDATA%\Android\Sdk` on Windows, `~/Library/Android/sdk` on macOS, `~/Android/Sdk` on Linux) or create `android/local.properties` with `sdk.dir=<absolute path>`.

### Development (live reload on a device)

```bash
npm run android:sync            # builds dist-native/ and copies into android/
npm run android:open            # opens android/ in Android Studio
# or:
npm run android:build           # plugs in a USB device → installs + launches
```

### Release APK / AAB

Open `android/` in Android Studio → **Build → Generate Signed Bundle / APK** → pick APK or Android App Bundle → create a keystore (first time) → choose `release` → build.

Outputs land under `android/app/release/`:

- `app-release.apk` — sideloadable on any Android 8.0+ device (~3 MB + WebView)
- `app-release.aab` — Google Play upload format

You can also produce a debug APK from the command line:

```bash
cd android
./gradlew assembleDebug       # macOS / Linux
gradlew.bat assembleDebug     # Windows
# APK appears at android/app/build/outputs/apk/debug/app-debug.apk
```

## Refreshing the bundled fonts

Google Fonts are self-hosted under `public/fonts/`. If you add a new weight or family, edit `scripts/fetch-fonts.ps1` (or write an equivalent shell script) and run:

```bash
npm run fetch-fonts
```

The script fetches the required woff2 files, deduplicates by content hash, and rewrites `public/fonts/fonts.css` to point at local copies.

## Repository layout

```
.
├── src/                  React + Vite app (shared across all targets)
├── public/               Static assets (fonts, logo) — copied to dist as-is
├── dist/                 Web build output  (singlefile HTML)
├── dist-native/          Native build output  (multi-file; used by Tauri + Capacitor)
├── src-tauri/            Tauri desktop shell (Rust + config)
│   ├── src/              main.rs + lib.rs — a thin shell around the webview
│   ├── tauri.conf.json   Window, bundle, and installer config
│   ├── Cargo.toml        Rust crate
│   └── icons/            Generated by `npx tauri icon`
├── android/              Capacitor Android project (openable in Android Studio)
├── capacitor.config.json Capacitor shared config (app id, webDir, plugins)
└── scripts/              Dev-time helpers (font fetching, etc.)
```

The Vite build reads `BUILD_TARGET` to switch between single-file (web) and multi-file (native) output. Everything else — including the interpreter, the canvas, the UI — is identical across targets.
